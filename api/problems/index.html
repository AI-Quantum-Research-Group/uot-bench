    uot.problems

    Problem classes, generators, and storage utilities.

    See Writing a custom Problem for a guide on subclassing.

    Base classes

    uot.problems.base_problem.Problem

    Bases: ABC

    Base class for all optimal-transport problems.

    Subclass this to define a custom problem and plug it into an :class:~uot.experiments.Experiment or :func:~uot.experiments.run_pipeline.

    Minimal subclass example::

    import jax.numpy as jnp
from uot import Problem
from uot.data import PointCloudMeasure

class MyProblem(Problem):
    def __init__(self, name, mu, nu, cost_fn):
        super().__init__(name, [mu, nu], [cost_fn])

    def get_marginals(self):
        return self.measures

    def get_costs(self):
        X, _ = self.measures[0].as_point_cloud()
        return [self.cost_fns[0](X, X)]

    def to_dict(self):
        return {"dataset": self.name}

    def free_memory(self):
        self._cost_cache = [None] * len(self.cost_fns)

    Abstract methods (must be implemented):

    • :meth:get_marginals — return list of :class:~uot.data.BaseMeasure
    • :meth:get_costs — return list of cost matrices as JAX arrays
    • :meth:to_dict — return a flat dict of metadata for DataFrame rows
    • :meth:free_memory — release any cached arrays

    Optional overrides:

    • :meth:get_lambdas — return barycenter weights (default: None)

    get_marginals() abstractmethod

    Return the marginal measures for this problem.

    get_costs() abstractmethod

    Return the cost matrices (as JAX arrays) for this problem.

    get_lambdas()

    Return barycenter weights, or None for two-marginal problems.

    post_solve_hooks()

    Return problem-level post-solve hooks.

    Override to attach domain-specific hooks (e.g. image reconstruction) that should run after every solve call on this problem. Each entry must match the :class:~uot.experiments.hooks.PostSolveHook protocol.

    to_dict() abstractmethod

    Return a flat dict of metadata to be stored as a DataFrame row.

    free_memory() abstractmethod

    Release any large cached arrays (cost matrices, etc.).

    uot.problems.problem_generator.Generator

    Bases: ABC

    Base class for problem generators.

    Subclass this to create a reusable source of :class:~uot.problems.Problem instances, then pass it to :func:~uot.experiments.run_pipeline.

    Minimal subclass example::

    from collections.abc import Iterator
from uot import Generator, TwoMarginalProblem
from uot.data import PointCloudMeasure
import numpy as np

class MyGenerator(Generator):
    def __init__(self, n: int, seed: int = 0):
        self.n = n
        self.rng = np.random.default_rng(seed)

    def generate(self) -> Iterator[TwoMarginalProblem]:
        from uot.utils.costs import cost_euclid_squared
        for _ in range(10):
            X = self.rng.standard_normal((self.n, 2))
            Y = self.rng.standard_normal((self.n, 2))
            mu = PointCloudMeasure(X, np.ones(self.n) / self.n)
            nu = PointCloudMeasure(Y, np.ones(self.n) / self.n)
            yield TwoMarginalProblem("my_problem", mu, nu, cost_euclid_squared)

    The generator stores its hyper-parameters in __init__; :meth:generate takes no arguments and yields problems one at a time.

    generate() abstractmethod

    Yield :class:~uot.problems.Problem instances.

    one()

    Return the first problem from this generator.

    solver_inputs(*, include_cost=True)

    Return :class:~uot.problems.SolverInputs for the first generated problem.

    point_cloud_inputs(*, shared_support='same', include_cost=True, include_zeros=True, atol=0.0, rtol=0.0)

    Return :class:~uot.problems.PointCloudInputs for the first generated problem.

    grid_inputs(*, include_cost=False, backend='auto', dtype=None, device=None)

    Return :class:~uot.problems.GridInputs for the first generated problem.

    Concrete problem types

    uot.problems.two_marginal.TwoMarginalProblem

    Bases: Problem

    get_costs()

    Returns a single‐element list containing the cost matrix between self._mu and self._nu, caching it in self._C so that repeated calls do not recompute.

    get_exact_cost()

    Return exact cost of transportation between measures self._mu and self._nu, caching it in the self._exact_cost, such that repeated calls do not recompute

    get_exact_coupling()

    Return exact transport plan between self._mu and self._nu, caching it so repeated calls do not recompute.

    to_ott_linear_problem(*, tau_a=1.0, tau_b=1.0, scale_cost=1.0, batch_size=None, epsilon=0.01)

    Return an OTT-JAX LinearProblem from this two-marginal problem.

    Requires pip install uot-bench[ott].

    to_ott_quadratic_problem(*, tau_a=1.0, tau_b=1.0, fused_penalty=0.0, scale_cost=1.0, epsilon=0.01, gw_unbalanced_correction=True, ranks=-1, tolerances=0.01)

    Return an OTT-JAX QuadraticProblem (GW) from this problem.

    Intra-space geometries are built from each measure's self-cost. Set fused_penalty > 0 to include the cross-space (fused GW) term. Requires pip install uot-bench[ott].

    uot.problems.multi_marginal.MultiMarginalProblem

    Bases: Problem

    Base class for multi-marginal OT problems.

    Subclass this (or :class:~uot.problems.Problem directly) and implement :meth:~uot.problems.Problem.get_marginals, :meth:~uot.problems.Problem.get_costs, :meth:~uot.problems.Problem.to_dict, and :meth:~uot.problems.Problem.free_memory.
